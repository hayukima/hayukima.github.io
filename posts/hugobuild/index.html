Hugo维护常用命令

1.在选定目录下生成博客网站

hugo new site 博客名

2.在post目录下生成一篇文章(放到统一文件夹下便于管理)

hugo new  post/article.md

3.本地调试操作(根目录下)

hugo server

4.初次部署到GitHub Pages上(在站点根目录执行)

#依次执行以下命令

hugo --theme=主题名 --baseURL="http://github用户名.github.io/"

cd public

git init

git add -A

git commit -m "xxxx"

git remote add origin https://github.com/github用户名/github用户名.github.io.git
# .git不能忘了

git push -f origin master

以上命令执行完毕后直接在浏览器里访问网站http://github用户名.github.io/即可。

4.部署完毕后的更新操作(五件套)

hugo

cd public

git add -A

git commit -m "xxxx"

git push -f origin master

当然，以上操作可以通过脚本来实现，省的每次发布新博客都要重写五行代码。

Markdown常用语法

见参考文档，随用随查，用多了便自然记住了，需要注意的是目前还不支持md文件内自定义html格式之类的，突出一个一切从简，返璞归真。

常见问题

1.后端修改在前端显示不及时：ctrl+F5清空浏览器缓存。

2.部署过程中的问题大部分集中在git的操作上，与博客本身没什么关系，包括但不限于：密钥的配置、与github建立远程连接、限制文件传输大小等等，及时参考网上资料便能解决。

3.更新搜索引擎：在根目录下运行

npm run index
